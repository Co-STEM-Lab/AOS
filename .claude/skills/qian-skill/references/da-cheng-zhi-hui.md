# 大成智慧（Da Cheng Zhi Hui）：人—机—网协同的工程版

钱学森晚年（1990 年代）提出的**大成智慧学**（the science of synthetic wisdom）是综合集成法的"形而上"升华。核心命题：

> **集大成、得智慧** —— 把人的经验、机器的算力、网络的知识结合起来，产生超过任何单一来源的智慧。

这与 AI 协同编码的本质完全同构。

## 三体系 + 一统合

钱学森把"综合集成研讨厅"具体化为四个要素：

```
专家体系（人）
        +
机器体系（计算 / AI / 自动化工具）
        +
知识体系（数据库 / 文献 / 代码库 / 经验沉淀）
        =
一统合：人机结合、人网结合、以人为主
```

**注意"以人为主" (human-led)**：钱学森反复强调，机器和网络是辅助，目的、价值判断、最终决定仍归于人。**这是对 AI 协同编码的根本约束**——不要把 AI 当成可以替代工程师的"自主决策者"。

## 软件工程版的"研讨厅"

把研讨厅落到 AI 编码：

| 钱学森原型 | AI 编码对应 | 角色 |
|---|---|---|
| 专家体系 | 工程师 + 领域专家 + 用户 + reviewer | 提出问题、判断价值、最终拍板 |
| 机器体系 | Claude / Copilot / 编译器 / LSP / profiler / CI / 形式化验证 | 加速搜索、生成、验证 |
| 知识体系 | 代码库 + 文档 + commit history + issue tracker + 内部 wiki + 外部论文 | 提供事实和先验 |
| 一统合 | Pair-coding session / PR review / architecture meeting | 三者循环协同 |

## 大成智慧的三个判据

钱学森提出，"得智慧"需要满足三件事：

1. **集大成**：要把多源信息融合，不偏废任何一方
2. **从定性到定量**：定性的洞察必须被定量证据验证或证伪
3. **人在回路 (human-in-loop)**：关键判断不能完全外包给机器

对照 AI 编码：

- **集大成失败**：只看 AI 生成的代码不读文档；只读文档不看实际运行数据
- **定量缺失**：AI 说"这样改更好"但不 benchmark / 不 profile / 不写测试
- **人不在回路**：把 PR 完全交给 AI auto-merge，没有人 review 决策意图

## 大成智慧 vs 单机智慧

钱学森的 insight 是：**单一来源的智慧有边界**。

- 纯人类工程师：受限于个人经验、记忆、注意力带宽
- 纯 AI：受限于训练数据截止时间、上下文窗口、无运行时观察
- 纯数据/网络：是死的，需要被解读

**只有三者协同**，才能在 OCGS / 复杂系统上做出"对"的判断。

## 实操：研讨厅模式的 AI 编码工作流

当你（AI）拿到一个复杂任务时，不要单跑。明确公开你处在哪个角色：

```
START
  |
  ├─ 步骤 1: AI = 知识检索员 (grep / 读代码 / 查 docs)
  |              → 把事实摆出来给人看
  |
  ├─ 步骤 2: 人 = 专家 (做价值判断 / 选方向 / 排除假设)
  |              → AI 不替人拍板
  |
  ├─ 步骤 3: AI = 验证执行员 (写测试 / 跑实验 / 量化)
  |              → 数据回到人手里
  |
  ├─ 步骤 4: 人 + AI = 协同综合 (讨论结果，迭代)
  |
  └─ 步骤 5: 人 = 最终决策者 (commit / deploy / 上线)
```

**关键反模式**：AI 在没有人类输入的情况下连续多步推理 + 自动执行高代价操作（部署、删数据、改 prod 配置）。这违背"以人为主"。

## 大成智慧与"科学"vs"经验"的辩证

钱学森还有一个重要观点：**经验**也是知识的一种，不能因为"不科学"就排斥。

- 老工程师的"我觉得这里有问题"是经验直觉，往往对
- AI 训练自海量代码，也是一种"经验"
- 形式化证明、benchmark 是"科学"

三者都不能独占。**大成智慧是把经验和科学集成的方法**。

## 与现代 AI agent 设计的呼应

大成智慧的设计原则，在现代 AI agent 架构里有直接映射：

| 钱学森 | 现代 AI agent |
|---|---|
| 专家体系 | Human-in-the-loop / Reviewer agents |
| 机器体系 | Tool use / function calling / MCP |
| 知识体系 | RAG / vector DB / context engineering |
| 以人为主 | Permission gates / approval workflows |
| 从定性到定量 | Self-consistency / verification / eval harness |
| 集大成 | Multi-agent collaboration / ensemble |

钱学森 1990 年的设想，30 年后被工程师重新发现并工程化。这本身就是大成智慧的活例子。
