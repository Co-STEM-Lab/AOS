# 现代科学技术体系四层结构

钱学森提出，任何成熟的科学领域应该有四层结构。这是他对**理论与实践如何衔接**的回答。

```
┌────────────────────────────┐
│   哲学层 (philosophy)       │   研究对象的本体、认识论、价值观
├────────────────────────────┤
│   基础科学 (basic science)  │   研究普适规律
├────────────────────────────┤  ← 桥梁 (bridge)
│   技术科学 (tech science)   │   把规律转化为可工程化的方法
├────────────────────────────┤
│   工程技术 (engineering)    │   具体实施
└────────────────────────────┘
```

**关键**：层之间需要**桥梁**——既不能让哲学直接指导施工，也不能让工程师只会拧螺丝不懂为什么。

## 软件工程的四层映射

| 层 | 内容 | 例子 |
|---|---|---|
| **哲学** | 编程哲学、设计哲学、价值判断 | Unix 哲学 / functional vs imperative / "code is liability" |
| **基础科学** | 计算理论、复杂性理论、信息论、形式语义 | Lambda calculus / 自动机理论 / PL theory / Computability |
| **技术科学** | 编译原理、数据库理论、分布式系统理论、网络协议 | CAP theorem / Raft / TCP / B+ trees |
| **工程技术** | 框架、库、工具、最佳实践 | React / Postgres / Kubernetes / git |

**为什么这个分层重要**：
- 跳层是常见错误。直接从"哲学（'微服务是好的'）"跳到"工程（'拆吧'）"，跳过技术科学（分布式系统理论），结果就是踩坑。
- 大多数工程师在工程技术层，少数在技术科学层，极少数在基础科学，几乎没有人系统在哲学层（除非你是 DHH / Linus / Knuth 级别）。**这没问题**——只要知道自己在哪一层，知道桥梁在哪。

## 本 Skill 的简化版（三层）

之前 SKILL.md 用的是简化的三层（哲学 / 基础理论 / 应用技术），合并了"基础科学"和"技术科学"。原因：

- 对绝大多数 AI 编码任务，**技术科学** 就够用，不需要回到 Turing 机原理
- 但对一些场景（设计新数据库、设计分布式协议、设计 ML 系统），**必须**回到基础科学

**何时该用四层完整版**：

- 你在设计一个新协议（不是用现有的）
- 你在解一个 NP-hard 优化问题（要知道复杂度边界）
- 你在做 ML 系统设计（要懂训练动力学、generalization theory）
- 你在做形式化验证（要懂 PL theory）
- 你在跨领域应用（如把控制论用到 autoscaler，把信息论用到 log 设计）

**何时三层就够**：日常 SWE 任务（CRUD、refactor、debug、性能优化、加 feature）。

## "桥梁"概念

钱学森反复强调，光有四层不够，关键是**层之间的桥梁**：

- 哲学 ↔ 基础科学：**方法论**（"如何做科学"、"如何做工程"）
- 基础科学 ↔ 技术科学：**应用数学 / 形式化建模**
- 技术科学 ↔ 工程技术：**设计模式 / 最佳实践 / 工程权衡**

软件工程的"桥梁"对应：

| 桥梁 | 软件版本 |
|---|---|
| 哲学 ↔ 基础 | "Why types matter" 类哲学文（Pierce, Wadler） |
| 基础 ↔ 技术 | 算法导论、TAOCP、Distributed Systems 教材 |
| 技术 ↔ 工程 | DDIA, Site Reliability Engineering, Google SRE books |

**这些是真正的"桥"——读它们可以帮你在层之间穿梭，而不是只在单一层重复劳动**。

## AI 编码场景的实操指南

每次任务前问自己：

1. **我现在在哪一层**？
2. **这个任务真的属于这一层吗，还是被错放在这一层**？
3. **我需要回到上一层吗**？（基础理论是否清晰？哲学/目标是否对齐？）

**典型错放**：
- "加个 feature 而已"（自以为在工程技术层）→ 实际涉及业务不变式（基础理论层）→ 不上去思考就直接撸代码 = 技术债
- "我们要用 ML 解决这个问题"（哲学层叫法）→ 跳过基础理论（这个问题用 ML 是否合适？数据够吗？） → 直接到工程（训模型）= 失败概率高
